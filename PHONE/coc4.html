Quality Inspection of Computer Systems

Steps to ensure a maintained computer is fully functional

💡
Key Principle

The effectiveness of checking maintained computers depends on the quality of inspection. A thorough inspection ensures the repaired system is safe and fully operational before returning it to the user.

⚡ Power On & Device Check

  1. 1
    Power On the Computer Press the power button and listen for POST (Power-On Self-Test) beeps. Confirm the system boots without error messages.
  2. 2
    Test All Computer Devices Verify all connected peripherals respond correctly.
  3. 3
    Check Monitor, Keyboard, and Mouse Confirm the display shows correctly, keyboard inputs are registered, and the mouse cursor moves smoothly.
  4. 4
    Check the Computer's Performance Open Task Manager or a performance tool to verify CPU, RAM, and Disk are operating at normal levels.
  5. 5
    Confirm the Computer is Performing at its Best No unusual lag, overheating, or error pop-ups during normal use.

🔌 Cable Inspection

⚠️
Check All Cable Connections

Inspect each cable type below and ensure they are fixed properly — no loose connectors, bent pins, or damaged insulation.

Cable TypeWhat to CheckCommon Issues
⚡ Power Cables Securely seated in PSU and motherboard connectors (24-pin, 8-pin CPU, SATA power) Loose connector causing no power or random shutdowns
🌐 LAN Cables RJ-45 connector clicks into port; no physical damage on the cable Crimped poorly, wrong wiring standard (T568A vs T568B)
🔗 USB Cables Peripheral devices recognized by the OS; no error in Device Manager Bent pins, faulty USB ports
🖥️ Monitor Cables HDMI/VGA/DisplayPort seated firmly; display shows correct resolution Flickering screen, no signal, wrong resolution

Computer-to-Computer Communications Testing

Verifying network connectivity and proper data sharing

🌐
Why Run Network Tests?

Running computer-to-computer communication checks ensures all computers in a network are connected properly for effective and easier file and information sharing.

📡 Pinging

💻
What is Ping?

The ping command is the basic tool for testing TCP/IP connectivity. It sends a special packet called ICMP Echo to a particular IP address and looks for a reply. If a reply is received, the network path to that device is working correctly.

⚙️ TCP/IP Settings — Key Terms

IP Address

A unique address assigned to a network adapter. A computer with multiple network adapters has one IP address for each, and each must be on a different subnet.

Subnet Mask

Used together with the IP address to determine which subnet an adapter belongs to. It separates the network portion from the host portion of the address.

Default Gateway

The IP address of a router or computer on the local area network that knows how to communicate with subnets not present on this computer.

DHCP Server

If an adapter is set to obtain an IP address automatically, the DHCP Server is the address of the server that provides it.

DNS Servers

IP address of one or more Domain Name Server computers. DNS translates human-readable domain names (e.g., google.com) into IP addresses.

How to Check TCP/IP

Open Command Prompt and type ipconfig /all to see all TCP/IP settings for each adapter.

🔥 Burn In & Bathtub Curve

Definition
🔥 What is Burn In?
Running a system or device for a period of time to ensure all components are working properly. The goal is to detect components likely to fail during the initial high-failure rate portion of the bathtub curve.
Definition
📉 What is the Bathtub Curve?
An equipment failure-rate curve with:
  1. Initial period — sharply declining failure rate (early failures)
  2. Middle period — prolonged constant-average failure rate (useful life)
  3. End period — failure rate increases sharply again (wear out)
🛁
Why It's Called "Bathtub"

The shape of the failure-rate curve — high at the start, flat in the middle, high at the end — resembles the cross-section of a bathtub! If Burn In is made sufficiently long (and artificially stressful), the system can be trusted to be mostly free of early failures afterward.

Report Making

Documentation after inspection and testing

What is a Report?

A piece of information describing or giving an account of certain events, given or presented to someone (e.g., a supervisor or client).

Why is a Report Important?

It enables you to practice skills of presentation, argument, evaluation, and calculation, and provides a permanent record of your work for accountability and future reference.

📋
What Should a Repair/Maintenance Report Include?
  • Date and time of work performed
  • Description of the problem or fault found
  • Actions taken (parts replaced, settings changed, tests run)
  • Test results (ping responses, device check status)
  • Recommendation for further action if needed
  • Technician's name and signature

Quick Reviewer — Key Concepts

Study these cards before your Written Test or Oral Questioning

Term
What does the ping command do?
Sends an ICMP Echo packet to an IP address and waits for a reply — used to test basic TCP/IP connectivity between devices.
Term
What is the purpose of a Subnet Mask?
Used with the IP address to determine which subnet an adapter belongs to — separates the network portion from the host portion.
Term
What is a Default Gateway?
The IP address of a router/computer on the local network that routes traffic to subnets not present on the local machine.
Term
What is DHCP Server?
The server that automatically assigns IP addresses to network adapters configured to receive one automatically.
Concept
What is Burn In testing?
Running a system under normal or stressful conditions for a period of time to detect early component failures before releasing it to the user.
Concept
What are the 3 phases of the Bathtub Curve?
1. Early failure (declining rate) → 2. Useful life (constant rate) → 3. Wear-out (rising rate again).
Concept
What cables must be checked during inspection?
Power Cables, LAN Cables, USB Cables, and Monitor Cables — all must be properly fixed with no damage.
Concept
What is the 5S principle?
Sort, Set in order, Shine, Standardize, Sustain — a workplace organization methodology to maintain safe and clean work environments.
Concept
What is WEEE?
Waste Electrical and Electronic Equipment directive — guidelines for the proper disposal of electronic components to minimize environmental harm.
Concept
What is the 3Rs waste management program?
Reduce, Reuse, Recycle — applied to electronic components and materials generated during computer maintenance and repair.
